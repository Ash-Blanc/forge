# Application & Use-Case Ideas Catalogue  
Leveraging Two-Stage UI Intent Decomposition  
(per-screen summarization → sequence intent extraction)

**Core method recap**  
- Stage 1: Small multimodal LLM (~7–8B) summarizes each UI interaction independently → structured text (screen context bullets + action description + optional sub-goal).  
- Stage 2: Lightweight model (same size, often fine-tuned) reads the ordered summaries and outputs concise overall intent (sentence or atomic facts).  
- Advantages: on-device, low latency (<100 ms/step), privacy (no raw screenshots leave device), cheap, scalable to long sessions, debuggable, outperforms same-size E2E/CoT and rivals large MLLMs.

Ideas are grouped by domain. Each entry specifies:  
- **Target** — users/environment  
- **Decomposed input** — what sequence is summarized in stage 1  
- **Stage 2 intent → value** — what it enables  
- **Why decomposition + small models wins** — key benefit

## 1. On-Device Personal Assistants & Mobile UX (Android/iOS, accessibility services)

1. **Task resumption & cross-app continuity**  
   Target: Smartphone users switching apps  
   Decomposed: Calendar → Gmail → Docs sequence + taps/switches  
   Stage 2: "Preparing meeting notes from email attachment" → auto-open draft with context pasted  
   Win: Privacy of personal workflow; arbitrary-length sessions without context blow-up

2. **Proactive navigation / route optimizer**  
   Target: Maps & ride-sharing users  
   Decomposed: Route preview scrolls, pin taps, time/filter changes  
   Stage 2: "Searching low-traffic evening route to airport" → suggest gas stops / voice ETA  
   Win: Offline, sub-100 ms latency

3. **Smart system settings guide**  
   Target: Elderly / novice users in Settings  
   Decomposed: Wi-Fi → Bluetooth → app toggles  
   Stage 2: "Pairing new headphones for music" → next-step popover / voice guidance  
   Win: Battery-friendly continuous monitoring

4. **Media playlist / watch-next builder**  
   Target: YouTube/Spotify users  
   Decomposed: Search → detail views → add-to-queue taps  
   Stage 2: "Building 80s synthwave playlist" → auto-queue similar tracks  
   Win: Listening history stays private

5. **Photo edit & share predictor**  
   Target: Social media users  
   Decomposed: Gallery crop → filter → caption steps  
   Stage 2: "Prepping beach vacation story with warm filter" → suggest caption/hashtags  
   Win: Personal photos never uploaded

6. **Shopping intent accelerator**  
   Target: E-commerce app users  
   Decomposed: Product views → cart adds → tab switches  
   Stage 2: "Comparing noise-cancelling headphones < $80" → price alerts / alternatives  
   Win: No behavioral data sent to servers

7. **Workout goal tracker**

   Decomposed: Exercise logs + progress taps  
   Stage 2: "Chasing monthly running distance PR" → adjust plan / recovery tips  
   Win: Always-on without draining battery

(Extendable to 50+ ideas — finance claim filing, gaming strategy helper, automotive distraction detector, gov form completer, etc.)

**Common implementation pattern template**  
Raw trajectory → Stage-1 JSON summaries per step → Stage-2 atomic intent facts → trigger (suggestion / macro / alert)

This approach unlocks production-grade, privacy-first, on-device intelligent agents across consumer and enterprise verticals.
